The deferred-delete FIFO

Overview 1 · Quiescent state 2 · QS variable 3 · Online / offline 4 · Deferred FIFO

Instead of hand-tracking every deleted pointer and its token, the writer just calls rcu_qsbr_dq_enqueue(). Each entry carries a grace-period token; the FIFO reclaims matured entries on its own.

Step 1 / 7

The matching code

Enqueue records the pointer with a fresh token and opportunistically reclaims; the reclaim pass walks the FIFO in order and stops at the first entry whose grace period has not closed.

At shutdown, rcu_qsbr_dq_delete() drains anything still pending — blocking on each token — and frees the FIFO itself.
