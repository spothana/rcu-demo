The quiescent state model

Overview 1 · Quiescent state 2 · QS variable 3 · Online / offline 4 · Deferred FIFO

Removing an element from a lock-free structure is split into two steps: delete (atomically unlink the reference) and free (return the memory, but only once no reader can still touch it). The window between them is the grace period.

The matching code

This is the writer loop from src/main.c. The atomic exchange is the delete; handing the old pointer to the FIFO defers the free until the grace period closes.

Reader 3 never references config A, so the writer does not wait on it — exactly the point that a critical section for one structure is a quiescent state for another.
