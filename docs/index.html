RCU QSBR — visual walkthrough

Four animations explaining the Quiescent State Based Reclamation (QSBR) RCU mechanism implemented in this repository. Each page pairs an animation with the matching code from src/rcu_qsbr.c and src/main.c. Every file is standalone HTML — open them directly, no server needed.

Animation 1

The quiescent state model

Delete and free split by a grace period; why a reader of a different structure is not waited on.

 Animation 2

The QS variable and the token

Per-reader counter slots; the writer issues a token and polls slots instead of talking to readers.

 Animation 3

Reader online and offline

A reader drops offline before a blocking call so the writer's check never stalls on it.

 Animation 4

The deferred-delete FIFO

Deleted pointers enqueued with a token, matured entries reclaimed back to the allocator.

How the animations map to the code

The walkthrough order matches the path a value takes through the library. Animation 1 is the writer in main.c doing the two-step delete/free. Animation 2 is rcu_qsbr_start() and all_passed() in rcu_qsbr.c. Animation 3 is rcu_qsbr_thread_offline() and the reader-0 path in main.c. Animation 4 is rcu_qsbr_dq_enqueue() and the reclaim pass.
