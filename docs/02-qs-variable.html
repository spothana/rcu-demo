The QS variable and the token

Overview 1 · Quiescent state 2 · QS variable 3 · Online / offline 4 · Deferred FIFO

The writer never talks to readers directly. It issues a token with rcu_qsbr_start(), then polls per-thread slots with rcu_qsbr_check(). A reader has passed token T once its slot counter reaches T.

The matching code

The QS variable is one global token plus a counter slot per reader. rcu_qsbr_start() bumps the token; all_passed() decides when the grace period is over.
